What Is Dataiku (DSS)?

Dataiku DSS (Data Science Studio) is an end-to-end data & AI platform — think of it as a workbench where you can bring your data, clean it, build models (including chatbots!), visualise results, and deploy — all in one place. You don't need to know every database or cloud tool; Dataiku creates a bridge for you.

🧹

Data Cleaning

Handle messy, incomplete or wrong values in your data before feeding it to any AI model.

📊

Visualisation

Build charts and dashboards to explore your data and share insights with your team.

🤖

ML & AI Models

Train machine learning models — or use LLMs / APIs — to make predictions or power chatbots.

🚀

Deployment

Bundle your finished project and move it to production with minimal effort — just bundle and click.

🔍

Monitoring

Watch how your deployed model performs over time and make adjustments as needed.

👥

Collaboration

Multiple team members — analysts, data scientists, engineers — can work on the same project without clashing.

Remember: Dataiku = DSS. The trainer used both names interchangeably throughout. They mean exactly the same tool.
Core Concepts You Must Know
🗂️ Projects

Everything in Dataiku lives inside a Project. When you create one, you give it a name (editable later) and a Project Key (permanent — never changes, used in the URL). For your chatbot, you'll create a new blank project.

Tip: Name your project clearly from the start, e.g. CHATBOT_YourName. The key can't be changed!
📋 Datasets (the blue squares)

A dataset is where your data lives. No matter whether it comes from Excel, PostgreSQL, S3, or an API — it always looks the same inside Dataiku. The icon changes to show the source, but the interface is identical. For a chatbot, your dataset might be a list of FAQs, documents, or conversation logs.

⚙️ Recipes (the circles) — Transformations

Recipes are the operations you perform on datasets to produce new datasets. Think: input data → recipe → output data.

🟡

Yellow = Visual Recipes

Click-and-configure, no coding needed. Includes: Join, Stack, Filter, Sort, Prepare, Group… These are the most common. Great for beginners!

🟠

Orange = Code Recipes

Write your own Python, R, or Scala code. Perfect for calling APIs (like an LLM/chatbot API) or doing custom logic.

🔴

Red = Plugins

Downloadable extensions for specific needs (e.g. Power BI connector, Geo plugin). Install only what you need from the Plugin Store.

Understanding the Flow

The Flow is the visual canvas of your project. Press G + F from anywhere to jump to it. It shows how your datasets and recipes connect — like a pipeline map.

📥 Source Data ──▶ 🟡 Recipe ──▶ 📋 Intermediate Dataset ──▶ 🟡 Recipe ──▶ ✅ Final Output
Navigation shortcut: Press G + F to go to Flow from anywhere in the app.
Day 1 Hands-On — What Was Done (Step by Step)

The class used a credit card fraud detection project as the training use case. Here's the exact sequence of steps. Understanding this order is key — it applies to any project, including your chatbot.

1

Create a New Project

Click + New Project → Blank Project. Set a name and a unique Project Key. The key is permanent — choose carefully.

2

Import Datasets

Go to Dataset → Upload your files to import CSV/Excel files. You can import multiple files at once and create separate datasets from them. The class imported 4 files: Transaction 2021, Transaction 2022, Card Holder Info, and Merchant Info.

3

Explore Your Data

Open any dataset to see its columns, data types, and Dataiku's smart suggestions. Use Analyse on a column to get stats (distinct values, % errors, min/max). Use Metrics to compute row count without loading everything.

Sampling: Dataiku shows you a sample (first 10,000 rows by default). Use random sampling for a better data picture. Compute whole-data stats separately when needed.
4

Stack Datasets (combine rows)

Select a dataset → click the Stack recipe. This appends one table on top of another (rows, not columns). Used to combine Transaction 2021 and 2022 into one table.

Stack ≠ Join. Stack adds rows vertically. Join adds columns horizontally.

5

Join Datasets (combine columns)

Select a dataset → click the Join recipe → add another dataset. Set the join condition (which column links both tables). The class did a Left Join to keep all transaction records even if card/merchant info was missing.

You can join more than 2 datasets in one recipe — just keep adding tables on the left side.

6

Prepare Recipe — Data Cleaning & Enrichment

The Prepare recipe (broom icon, yellow) is your Swiss Army knife. You add steps inside it — each step transforms the data. Steps in Day 1:

  • Remove invalid rows — deleted error records in the "Signature Provided" column
  • Find & Replace — created a new "Flag Description" column mapping 0→Fraud, 1→Authorised
  • Create Geopoint — combined latitude + longitude into a single map point for card holder and merchant locations
  • Reverse Geocode — turned geopoints into state names (using a plugin)
  • Switch (case logic) — created "Spending Type" column categorising merchant sectors
  • If-Then-Else — created a "Risk" column (High / Medium / Low) based on purchase amount + signature
Pro tip: Don't run after every step. Add ALL your steps first, then run once at the end. This is much faster and lets you catch mistakes early.
7

Run & Verify

Click Run to execute the full recipe. Check the output dataset — verify row count and confirm all new columns were created as expected.

🤖 Your Chatbot Project Roadmap in Dataiku

Here's how to map what you learned today to building your chatbot in Dataiku:

📁

Step 1 — Create Your Project

New project → give it a name like CHATBOT_YourName. This is your workspace.

📄

Step 2 — Import Your Knowledge Data

Upload the documents, FAQs, or text files your chatbot should know about. These become datasets. CSV, Excel, or TXT all work.

🧹

Step 3 — Clean & Prepare Text Data

Use the Prepare recipe (yellow) to remove duplicates, fix formatting, trim whitespace, or split long documents into chunks. Clean text = better chatbot answers.

For chatbots: You may need to split documents into smaller paragraphs. Use the Split processor inside Prepare.
🔌

Step 4 — Call an LLM API (Orange / Code Recipe)

Use an Orange code recipe (Python) to call an LLM like OpenAI's GPT or Anthropic's Claude. Pass your cleaned text chunks + user questions. Return answers as a new column.

As the trainer explained: anything you can do in Python can be done inside Dataiku. Web scraping, API calls, pandas — all work inside a code recipe. The output gets stored as a dataset.
📊

Step 5 — Visualise & Test (Tomorrow's topic)

Use charts to explore your data patterns. Day 2 will cover this — you'll see how to create visualisations in Dataiku.

🚀

Step 6 — Deploy

Once happy, bundle your project and deploy it. Dataiku handles the infrastructure. Your chatbot pipeline becomes a live service.

User Types & Licenses

Admin License

Full access. Sets up the instance, manages users and connections. Free with designer seats.

🧑‍💻

Designer License

The paid seat. Used for development — building flows, recipes, models. This is what you use in training.

Free. View-only — can see dashboards, flows, and datasets. Business users get this. 1 designer = 10 viewers.

Computation Engines (Good to Know)

Dataiku automatically picks the best engine to run your recipes. You don't need to worry about this as a beginner, but it's good context:

🖥️
DSS Local — Default. Dataiku runs the computation itself. Good for most tasks.
🗄️
SQL Engine — Pushes the query to your database (e.g. SQL Server) to run it there — often much faster for large data.
⚡
Spark — Distributed computing for huge datasets. Only available if your company has Spark infrastructure.
Rule of thumb: If your data is in a SQL database, switch the engine to SQL — it'll be significantly faster.
Common Questions from the Session
Can I scrape websites or call APIs to get data?
Yes! The trainer confirmed this. Open an Orange code recipe (Python), write your scraping or API-calling code, and store the result in a dataset. It works with pandas DataFrames — just like regular Python. Your chatbot can call an LLM API this exact way.
What's the difference between Stack and Join?
Stack = adds rows vertically (same columns, more data). Think: appending two months of transaction data into one table.

Join = adds columns horizontally (linking two tables by a shared key). Think: enriching your transaction table with customer details.
What are the 3 types of Joins and when do I use them?
Left Join — Keep ALL rows from the left table; only bring matching rows from the right. If no match, fill with NULL. ✅ Most common, use this by default.

Inner Join — Only keep rows where there's a match in BOTH tables. Non-matching rows disappear.

Full Outer Join — Keep everything from both tables, filling NULLs where there's no match.
What does Sampling mean and why does it matter?
Dataiku can't load millions of rows into the UI instantly, so it shows you a sample (default: first 10,000 rows). Always switch to Random sampling for a more representative view. To compute exact row counts, use the Metrics → Compute button or click the refresh icon in the dataset header.
What's the difference between Remove, Clear, and Filter in Prepare?
Remove — Deletes the entire row. Your record count decreases.

Clear — Empties the cell value (makes it NULL) but keeps the row. Row count stays the same.

Filter — Keeps only rows matching a condition; everything else is removed.
How do I share my project with a teammate?
Go to your project → Security → Add their Dataiku user ID → Choose access level (Read, Write, Admin). Usually teams are organised into groups to simplify this.
