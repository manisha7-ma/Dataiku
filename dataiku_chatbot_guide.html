B

Ask Your Admin to Set Up the LLM Connection

⚙️ Admin Panel → Connections → LLM Mesh

Dataiku uses something called LLM Mesh — it's the bridge between Dataiku and AI models like ChatGPT. Only an admin can set this up. You don't do this yourself — just ask your admin to do it and give you access.

What your admin needs to do
1️⃣ Go to Administration → Connections → New Connection → LLM Mesh section
2️⃣ Select OpenAI (easiest for beginners). Enter the OpenAI API Key (get one at platform.openai.com). Enable GPT-4o model.
3️⃣ Click Test to confirm the connection works. Click Save.
4️⃣ Share the connection name with you (e.g. openai-gpt4). You'll need this name later.
Tip: If your company uses Azure OpenAI or Anthropic Claude, those work too. The steps in this guide are the same — just the connection name differs.

3

Import Your SQL Table as a Dataset

📁 Your Project → + Dataset → SQL Databases

Now bring your actual table data into Dataiku. A dataset in Dataiku is a table — it can come from a SQL database, a file, or anywhere. Once imported, it looks the same regardless of source.

1️⃣ Inside your project, click + Dataset (in the top menu or the Flow). Under SQL Databases, click your connection name (e.g. my_postgres_db).
2️⃣ Dataiku shows you all available tables in that database. Click the table you want (e.g. customers).
3️⃣ A preview of the data appears. Check it looks correct (column names, sample data). Click Create Dataset.
4️⃣ Repeat for every table your chatbot needs to know about (e.g. orders, products).
What just happened? Dataiku created a pointer to your SQL table. It doesn't copy all your data — it reads it live from the database. The blue square icon in the Flow represents this dataset.
PHASE 2

Build Your Data Flow — Clean & Prepare

Before the AI can answer questions about your data, you need to clean it and add helpful descriptions. Think of it like labelling boxes before storing them — the AI understands labelled data much better.

4

Go to the Flow and Verify Your Datasets

📋 Project → Flow (or press G + F)
⌨️ Press G + F on your keyboard to jump to the Flow at any time. This is your project's visual workspace — you'll spend most of your time here.
👁️ You should see your table(s) as blue squares (datasets). Click any dataset to open it and preview the data — check the columns look right.
📊 Click the Columns tab inside a dataset. Dataiku shows you every column name and what data type it detected (string, integer, date, etc.). Also click Analyze on any column to see stats — % of empty values, distinct values, etc.
Why check this? If a column has lots of empty values or incorrect data types, the AI will give poor answers. You want clean, well-structured data going in.
5

Add Column Descriptions (Metadata)

📋 Dataset → Schema tab → Edit column descriptions

This is one of the most important steps for a good chatbot. When you add descriptions to your columns, the AI understands what each column means — not just its name. For example, a column named cust_id becomes clear when you add "Unique customer identifier". Without descriptions, the AI often makes wrong guesses.

1️⃣ Open your dataset → click the Schema tab (or Columns tab depending on your DSS version).
2️⃣ For each column, click into the Comment/Description field (usually to the right of the column name) and type a plain-English description.
3️⃣ Click Save Schema when done.
Column NameExample Description to Add
cust_idUnique numeric ID for each customer
cust_nameFull name of the customer
order_dateDate the order was placed (YYYY-MM-DD format)
order_totalTotal value of the order in USD
statusOrder status: pending, shipped, or delivered
AI shortcut (DSS 13+): In the Schema tab, look for a "Generate Metadata with AI" button. Click it and Dataiku will auto-generate descriptions using the LLM — then you just review and correct them. This saves a lot of time!
6

Create a Prepare Recipe to Clean Your Data

📋 Flow → Click Dataset → + Recipe → Prepare

Even if your data looks clean, there are small issues the AI will struggle with: empty cells, wrong date formats, columns with many nulls. The Prepare recipe (yellow broom icon) is where you fix all of this visually — no coding needed.

1️⃣ In the Flow, click on your dataset (blue square). A side panel opens. Click + Recipe, then select Prepare (yellow broom icon).
2️⃣ Dataiku asks for an output dataset name. Type something like customers_prepared. Click Create Recipe.
3️⃣ The Prepare recipe editor opens. You'll see your data on the right. Common cleaning steps to add (click + Add New Step):
🗑️ Remove rows with empty values: Step → Filter → Remove rows where [column] is empty. Use this for critical columns like customer ID or order date that must have values.
✂️ Trim whitespace: Step → String → Trim whitespace from [column]. Removes invisible spaces at the start/end of text that cause matching problems.
🔠 Normalize text case: Step → String → Lowercase [column]. Makes text comparisons reliable (e.g. "Pending" and "pending" become the same).
📅 Fix date columns: Click a date column → Parse date. Tell Dataiku the format (e.g. YYYY-MM-DD). This lets the AI answer "orders in the last 30 days" type questions.
4️⃣ Once done adding steps, click Run (top right). Wait for the green checkmark. Your clean dataset is now ready.
Remember: Don't run after each individual step. Add all your steps first, then run once at the end. It's much faster, and you can see the preview of what each step will do on the right side panel before running.
7

Join Multiple Tables (if you have more than one)

📋 Flow → Select 2 datasets → + Recipe → Join

If your chatbot needs to answer questions about data that lives in different tables (e.g. "Which customers have pending orders?"), you need to join those tables. A Join recipe combines two tables into one using a shared column (like customer ID).

1️⃣ In the Flow, click on your primary dataset (e.g. customers_prepared). In the side panel, click + Recipe → Join.
2️⃣ Click Add a dataset and select the second table (e.g. orders). Name the output something like customers_with_orders.
3️⃣ Set the Join condition: pick the shared column from each table. For example: customers.cust_id = orders.customer_id.
4️⃣ Choose Left Join (keeps all customers even if they have no orders). Click Save then Run.
Left Join vs Inner Join: Left Join = keep all rows from the left table, add matching data from right (nulls if no match). Inner Join = only keep rows where both tables have matching data. Use Left Join when you don't want to lose rows.

Write the Python Code — Full Explained Version

📋 Python Recipe Editor

chatbot_knowledge_base Python Recipe — Full Code PYTHON
# ═══════════════════════════════════════════════════════════
# CHATBOT KNOWLEDGE BASE BUILDER
# This script:
#   1. Reads your cleaned SQL data from Dataiku
#   2. Converts each row into a human-readable text summary
#   3. Stores those summaries so the chatbot can use them
# ═══════════════════════════════════════════════════════════

# --- IMPORTS ---
# These are libraries (toolkits) we need.
# 'dataiku' lets us talk to Dataiku from Python.
# 'pandas' is the standard Python library for working with tables (DataFrames).
import dataiku
import pandas as pd

# --- STEP 1: READ YOUR INPUT DATASET ---
# 'dataiku.Dataset()' connects to a dataset inside your Dataiku project.
# Replace "customers_with_orders" with your actual dataset name if different.
# .get_dataframe() loads it into a pandas DataFrame — a table in Python memory.
input_dataset = dataiku.Dataset("customers_with_orders")
df = input_dataset.get_dataframe()

# --- STEP 2: PREVIEW YOUR DATA ---
# This prints the first 3 rows so you can confirm the data loaded correctly.
# Remove this line after testing.
print("Data loaded. Shape:", df.shape)
print(df.head(3))

# --- STEP 3: CONVERT EACH ROW TO A TEXT SUMMARY ---
# The chatbot works with TEXT, not raw table data.
# We need to turn each row (record) into a sentence that describes it.
# This function takes one row and turns it into a text description.
#
# IMPORTANT: Customize this function based on YOUR column names!
# Replace 'cust_name', 'order_date', 'order_total', 'status'
# with the actual column names in your dataset.
# Check your dataset's Columns tab in Dataiku to find your column names.

def row_to_text(row):
    """
    Converts a single row of data into a readable text summary.
    The chatbot uses these texts to answer user questions.
    
    Customize this based on your data!
    """
    # Build the summary sentence by sentence.
    # f"..." is an f-string — it lets you insert variable values into text.
    # row['column_name'] gets the value from a specific column for that row.
    summary = (
        f"Customer: {row['cust_name']}. "
        f"Customer ID: {row['cust_id']}. "
        f"Order placed on: {row['order_date']}. "
        f"Order total: ${row['order_total']}. "
        f"Order status: {row['status']}."
    )
    return summary

# Apply the function to every row in the DataFrame.
# .apply() runs the function on each row and creates a new column called 'text_summary'.
df['text_summary'] = df.apply(row_to_text, axis=1)

# --- STEP 4: KEEP ONLY WHAT'S NEEDED ---
# For the knowledge base, we only need the text summary and a unique ID.
# Add any other columns that help identify the record (e.g., cust_id, order_id).
output_df = df[['cust_id', 'text_summary']].copy()

# --- STEP 5: WRITE OUTPUT TO DATAIKU ---
# This saves our new dataset (with text summaries) back into the Dataiku Flow.
# 'chatbot_knowledge_base' must match the output dataset name you created
# when setting up this recipe in Step 9.
output_dataset = dataiku.Dataset("chatbot_knowledge_base")
output_dataset.write_with_schema(output_df)

print("✅ Done! Knowledge base created with", len(output_df), "records.")
Before running: Replace the column names (cust_name, order_date, etc.) with the actual column names from your dataset. Open your dataset → Columns tab to see the exact names. They must match exactly, including upper/lowercase.
5️⃣ Click Run (top right of the recipe editor). Watch the logs at the bottom. You should see "✅ Done! Knowledge base created with X records." If there's an error, the most common cause is a column name mismatch.

Create the AI Query Function — Making the Chatbot Intelligent

📋 Flow → New Python Recipe (input: knowledge_base, output: query_results)

This optional but powerful step creates a recipe that can take a question, search your knowledge base for relevant records, send them to the AI, and return an answer. This is called RAG — Retrieval Augmented Generation. Think of it as giving the AI a cheat sheet of your data before asking it a question.

📌 What is RAG? Instead of just asking the AI a question (which it might not know the answer to), you first find the most relevant rows from your database, then include those rows as "context" in the question. The AI then answers based on the real data — not guesswork.
RAG Query Function — Full Explained Code PYTHON
# ═══════════════════════════════════════════════════════════
# RAG QUERY RECIPE
# This recipe:
#   1. Takes sample questions
#   2. Finds relevant rows from the knowledge base
#   3. Sends those rows + the question to the AI
#   4. Returns the AI's answer
# ═══════════════════════════════════════════════════════════

import dataiku
import pandas as pd

# --- CONFIGURATION ---
# Replace with your actual LLM ID (from Step 8)
LLM_ID = "openai:my-openai-conn:gpt-4o"  # ← CHANGE THIS

# --- READ THE KNOWLEDGE BASE ---
# This is the dataset with all the text summaries we created earlier.
kb_dataset = dataiku.Dataset("chatbot_knowledge_base")
kb_df = kb_dataset.get_dataframe()

# --- SET UP THE LLM ---
client = dataiku.api_client()
project = client.get_default_project()
llm = project.get_llm(LLM_ID)


# ─────────────────────────────────────────────────
# FUNCTION: search_knowledge_base
# Purpose: Find relevant rows from the knowledge base
# How it works: Simple keyword search — looks for the
#   question's words in the text summaries.
# ─────────────────────────────────────────────────
def search_knowledge_base(question, kb_df, max_results=5):
    """
    Searches the knowledge base for rows relevant to the question.
    Returns the top matching rows as a list of text strings.
    
    Parameters:
        question: The user's question (a string)
        kb_df: The knowledge base DataFrame
        max_results: How many matching rows to return (default 5)
    """
    # Convert question to lowercase for case-insensitive matching
    question_lower = question.lower()
    
    # Split question into individual words to search for
    keywords = question_lower.split()
    
    # Score each row by how many keywords it contains
    def score_row(text):
        text_lower = str(text).lower()
        return sum(1 for kw in keywords if kw in text_lower)
    
    kb_df = kb_df.copy()
    kb_df['_score'] = kb_df['text_summary'].apply(score_row)
    
    # Sort by score (highest first) and take top results
    top_rows = kb_df.sort_values('_score', ascending=False).head(max_results)
    
    # Return as a list of text summaries
    return top_rows['text_summary'].tolist()


# ─────────────────────────────────────────────────
# FUNCTION: ask_ai
# Purpose: Send the question + relevant data to the AI
# ─────────────────────────────────────────────────
def ask_ai(question, context_rows):
    """
    Sends the question and relevant data rows to the AI and returns the answer.
    
    Parameters:
        question: The user's question
        context_rows: List of relevant text summaries from the knowledge base
    """
    # Join all the context rows into one block of text
    context_text = "\n".join(context_rows)
    
    # Build the full prompt.
    # The system message tells the AI its role.
    # The user message includes the real data + the question.
    system_message = """You are a helpful data analyst assistant. 
You are given data records from a database.
Answer questions based ONLY on the provided data.
If the answer is not in the data, say 'I don't have that information in the database.'
Keep answers clear and concise."""
    
    user_message = f"""Here are the relevant records from the database:

{context_text}

Based on this data, please answer the following question:
{question}"""
    
    # Create and send the request to the AI
    completion = llm.new_completion()
    completion.with_message(system_message, role="system")
    completion.with_message(user_message, role="user")
    
    # Execute and return the response text
    response = completion.execute()
    return response.text


# ─────────────────────────────────────────────────
# MAIN: Run sample questions
# Customize these questions based on your data!
# ─────────────────────────────────────────────────
sample_questions = [
    "Who are the top 3 customers by order total?",
    "How many orders are currently pending?",
    "What is the average order value?",
]

results = []

for question in sample_questions:
    print(f"\n📨 Question: {question}")
    
    # Step 1: Find relevant rows
    relevant_rows = search_knowledge_base(question, kb_df)
    print(f"   Found {len(relevant_rows)} relevant records")
    
    # Step 2: Ask the AI
    answer = ask_ai(question, relevant_rows)
    print(f"🤖 Answer: {answer}")
    
    # Step 3: Store the result
    results.append({
        'question': question,
        'answer': answer,
        'num_records_used': len(relevant_rows)
    })

# --- SAVE THE RESULTS ---
# This saves the Q&A pairs to a new dataset in Dataiku.
# Useful for reviewing what the chatbot answered.
results_df = pd.DataFrame(results)
output_dataset = dataiku.Dataset("query_results")
output_dataset.write_with_schema(results_df)

print("\n✅ Done! Results saved to query_results dataset.")
After running: Open the query_results dataset in the Flow to review what the AI answered. This helps you tune the questions and system message before building the full chatbot UI.

18

Keep Data Fresh — Set Up Automatic Refresh

📋 Flow → Scenarios → New Scenario

Right now, your chatbot only knows about the data that existed when you ran the recipes. If new orders or customers are added to your SQL database, the chatbot won't know until you re-run the recipes. Scenarios automate this.

1️⃣ In your project top menu, click Scenarios → + New Scenario.
2️⃣ Click Add Step → Build / Train → Build Dataset. Select chatbot_knowledge_base. Check "Recursive" — this rebuilds everything in the pipeline automatically.
3️⃣ Click Add Trigger → Time-based. Set it to run every day at 6 AM (or whatever makes sense for your use case). Save the scenario.
Result: Every morning at 6 AM, Dataiku automatically re-reads your SQL database, re-runs the Prepare recipe and Python recipe, and updates the knowledge base. Your chatbot always has fresh data without you doing anything manually.

Quick Glossary
LLM
Large Language Model — the AI brain (e.g. GPT-4o, Claude). It reads text and generates answers.
LLM Mesh
Dataiku's central hub for managing AI connections. Admin sets it up once, everyone uses it.
RAG
Retrieval Augmented Generation — find relevant data first, then send it to the AI with the question.
Dataset
A table in Dataiku — can come from SQL, files, APIs etc. Always looks the same once imported.
Recipe
A transformation step in the Flow. Yellow = visual, Orange = code (Python), Red = plugin.
Flow
The visual pipeline canvas. Shows all datasets and recipes connected together. Press G+F to open.
Knowledge Base
The prepared dataset that the chatbot searches through to find relevant information.
Answers Plugin
A Dataiku plugin that gives you a ready-made chat UI — no front-end coding needed.
System Prompt
Instructions you give the AI about how to behave — its "personality" and rules.
Scenario
An automated workflow in Dataiku that runs recipes on a schedule (like a cron job).
DataFrame
A table loaded into Python memory using the pandas library. Like an Excel sheet in code.
Project Key
Permanent unique ID for your project — shown in the URL. Never changes after creation.
